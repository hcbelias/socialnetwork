Introduction

What is MongoDB?

  • First developed by the software company 10gen (now MongoDB Inc.) in October 2007, shifted to an open source development model in 2009
  • Written in C++
  • MongoDB (from humongous) is a cross-platform document-oriented database
  • Relational database stores data in tables, MongoDB stores data as documents, which are JSON-like field and value pairs

Why MongoDB?

  • Performance and Scalability
  • Schemaless
  • No SQL

Database Operations

Query

In MongoDB a query targets a specific collection of documents. Queries specify criteria, or conditions, that identify the documents that MongoDB returns to the clients. Query may include a projection that specifies the fields from the matching documents to return. You can optionally modify queries to impose limits, skips, and sort orders.
  • Query using Modifier
  • Query using Projection

Data Modification

Data modification refers to operations that create, update, or delete data. In MongoDB, these operations modify the data of a single collection. For the update and delete operations, you can specify the criteria to select the documents to update or remove.

Operations

INSERT

One document

db.users.insert({ 'name': 'henrique', 'email': 'helias@avenuecode.com'});
WriteResult({ "nInserted" : 1 });

Multiple documents

db.users.insert([
   { 'name': 'henrique', 'email': 'helias@avenuecode.com'},
  { 'name': 'luiz', 'email': 'luiz@avenuecode.com', 'age': 31},
  { '_id': 999, 'name': 'jose', 'email': 'jose@avenuecode.com', 'age': 30}
]);
BulkWriteResult({
  "writeErrors" : [ ],
  "writeConcernErrors" : [ ],
  "nInserted" : 3,
  "nUpserted" : 0,
  "nMatched" : 0,
  "nModified" : 0,
  "nRemoved" : 0,
  "upserted" : [ ]
});

If the collection does not exist, then the insert() method will create the collection.

If the document does not specify an _id field, then MongoDB will add the _id field and assign a unique ObjectId for the document before inserting. If the document contains an _id field, the _id value must be unique within the collection to avoid duplicate key error.

NodeJS Driver

Usually uses the same syntax from MongoDB shell.


MongoDB mongo mydatabase
db.users.find({ 'name' : 'henrique' });

 NodeJS var MongoClient = require('mongodb').MongoClient;
MongoClient.connect('mongodb://localhost:27017/mydatabase', function(err, db) {
if(err) throw err;
var query = { 'name' : 'henrique' };
var cursor = db.collection('users').find(query);
cursor.each(function(err, doc) {
   if(err) throw err;
   if(doc == null) {
     return db.close();
   }
   console.dir(doc.name);
   });
});

Query Filters

Conditions

  • All Documents db.users.find({})
  • Specify Equality Condition db.users.find( { name: 'henrique'} )
  • Specify AND Conditions db.users.find( { name: 'henrique', email: 'helias@avenuecode.com' } )
  • Specify OR Conditions db.users.find( { $or: [ { name: 'henrique'}, { name: 'helias@avenuecode.com'}] })
  • Specify AND as well as OR Conditions db.users.find( { $or: [ { name: 'henrique'}, { name: 'helias@avenuecode.com'}], age: 31 } )

Embedded Documents

When the field holds an embedded document, a query can either specify an exact match on the embedded document or specify a match by individual fields in the embedded document using the dot notation.

  • All Documents db.users.find({})
  • Specify Equality Condition db.users.find( { name: 'henrique'} )
  • Specify AND Conditions db.users.find( { name: 'henrique', email: 'helias@avenuecode.com' } )
  • Specify OR Conditions db.users.find( { $or: [ { name: 'henrique'}, { name: 'helias@avenuecode.com'}] })
  • Specify AND as well as OR Conditions db.users.find( { $or: [ { name: 'henrique'}, { name: 'helias@avenuecode.com'}], age: 31 } )
